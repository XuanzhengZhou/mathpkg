This theorem states that Kolmogorov complexity is algorithmically undecidable: there is no effective procedure to compute the complexity $C_u(f)$ of a given recursive function $f$. The intuitive reason is that if complexity were computable, one could diagonalize against the enumeration to construct a function whose complexity would violate the optimality bounds. Combined with the Kolmogorov theorem on optimal families, this result shows that complexity provides a well-defined but fundamentally non-constructive measure of the information content of recursive functions.
