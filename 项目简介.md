# mathpkg — 数学知识图谱与形式化验证系统

> 2026年6月 · 个人研究项目

## 一、项目概述

**mathpkg** 是一个数学知识工程系统，目标是将 Springer 出版社 1,565 本数学教材中的每一个定理、定义、引理提取出来，构建一个可检索、可验证、有依赖关系的数学知识图谱。

核心理念：**数学知识管理等价于软件包管理**。我们直接复制 apt 包管理器的架构，将数学概念视为"包"，概念间的依赖关系视为"包依赖"，使用拓扑排序生成学习路径。

## 二、核心能力

### 1. 大规模概念提取

- 从 OCR 后的教材页面中，使用 DeepSeek Pro 大模型自动识别和提取所有数学概念
- 每个概念输出三个文件：`concept.yaml`（元数据+依赖）、`theorem.tex`（纯 LaTeX 数学陈述）、`introduce.en.md`（自然语言解读）
- 目前已从 10 本测试书中提取 **6,374 个概念**，平均 ¥8/书

### 2. Section 级自动拆分

- 开发了一套四级自动拆分策略（正则表达式 → 章节检测 → Flash 语义分析 → 大小回退）
- 发现数学教材的**页/section ≈ 10 的自然规律**，跨书极其稳定（短书 8 页，长书 12 页）
- 131 本书拆分出 4,482 个 section，94% 自动化成功率，总成本仅 ¥4

### 3. Mathlib4 复用

- 构建了 Lean 4 定理证明器标准库 Mathlib4 的完整索引：**204,000 条**记录，含名称、文件路径、类型签名和文档注释
- 设计了三级倒排索引匹配策略，概念到 Mathlib4 定理的自动匹配率达 **71.6%**
- 精确名称匹配几乎无效（1%），签名关键词+数学域过滤才是核心方法（69%）

### 4. 本地 GPU OCR 管线

- 使用 ZhipuAI GLM-OCR 模型在本地 RTX 5060Ti 上运行
- 逐页保存 + 即时 checkpoint，支持断点续跑
- 处理速度 7.4 本/小时，已处理 171 本/60,000+ 页（1,565 本全部完成预计 8 天）
- 内置 CUDA OOM 自动回退机制和每小时健康检查

## 三、技术架构

```
Springer PDF 教材 (1,565 本)
    ↓ [GLM-OCR, 本地 GPU]
逐页 OCR JSON
    ↓ [Section 拆分: regex → chapter → Flash → size]
Section .md 文件 (~10 页/个, 共 4,482 个)
    ↓ [DeepSeek Pro Agent, 每 section 一个, 并行]
概念文件 (concept.yaml + theorem.tex + introduce.en.md)
    ↓ [去重 + 依赖消解]
知识图谱 (graph.db, 仿 apt pkgCache)
    ↓ [Mathlib4 倒排索引匹配]
Lean 4 形式化验证代码
```

## 五、关键发现

1. **页/section ≈ 10**：这一自然规律在 129 本教材中验证，为 Agent 处理的输入大小提供了理论依据
2. **Section > Chapter**：章节（30-60 页）对模型过大易截断，section（~10 页）是提取的最佳粒度
3. **倒排索引 > 精确 grep**：在 Mathlib4 匹配中，精确名称匹配仅命中 1%，而签名关键词+域过滤命中 69%
4. **OOM 回退是必需的**：本地 GPU 显存波动导致 ~10% 的书出现 CUDA OOM，batch=8→2 的自动回退机制解决了这一问题
