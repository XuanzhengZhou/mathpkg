# Pipeline V4 — 完整规格 (2026-06-24)

## 核心理念

每个概念 = 一个文件夹。每个文件 = 一种角色。YAML frontmatter 声明一切。

```
concepts/group_theory/lagrange_theorem/
├── theorem.md           ← primary: 定理陈述 (LaTeX + 自然语言)
├── proof_gtm0073.md     ← secondary: Hungerford 证明 (attaches_to: theorem.md)
├── proof_gtm0080.md     ← secondary: Robinson 证明 (不同讲法)
├── proof_gtm0148.md     ← secondary: Rotman 证明
├── subgroup_def.md      ← primary: 子群定义 (predecessor dependency)
├── coset_def.md         ← primary: 陪集定义 (predecessor dependency)
├── exercises.md         ← secondary: exercises (attaches_to: theorem.md)
├── verify.lean          ← secondary: Lean 形式化 (attaches_to: theorem.md)
├── introduce.md         ← secondary: 自然语言导读 (attaches_to: theorem.md)
├── _manifest.json        ← auto-generated by Python (never hand-edit)
└── _compiled.md          ← auto-generated encyclopedia page
```

**每个 .md 文件自带 YAML frontmatter。** 不维护中央 YAML。Python 脚本扫描目录自动生成 `_manifest.json`。

## Step 0: YAML Frontmatter 规范

每个 .md 文件以 YAML frontmatter 开头，用 `---` 分隔：

### Primary 节点 (theorem, lemma, proposition, corollary, definition, axiom)

```yaml
---
role: theorem
id: lagrange_theorem
name: {en: "Lagrange's Theorem", zh: "拉格朗日定理"}
topic: group_theory
difficulty: basic

source:
  book: gtm-0073
  chapter: I
  section: I.4
  pages: 38-42
  theorem_number: "Theorem 4.6"
  author: Hungerford

attachments:
  - introduce.md       # 自然语言解读
  - proof_gtm0073.md   # 本教材证明
  - exercises.md       # 配套习题
  - verify.lean        # Lean 形式化

depends:
  required: [subgroup, coset]
  recommended: [group_action]
---
# Lagrange's Theorem

Let G be a finite group and H ≤ G. Then |H| divides |G|...
```

### Secondary 节点 (introduce, proof_X, exercises, verify.lean)

```yaml
---
role: proof
attaches_to: theorem.md
source:
  book: gtm-0080
  author: Robinson
  approach: index_multiplicativity
---
## Proof (Robinson, via Index Multiplicativity)

1. By the Multiplicativity of Index,...
```

### 规则

1. **Primary 节点** (theorem, definition, axiom, lemma, proposition, corollary) 可以有 attachments
2. **Secondary 节点** (introduce, proof, exercises, verify.lean) 必须有 `attaches_to`，指向 primary 节点
3. `introduce.md` 和 `verify.lean` 永远是 secondary
4. `proof_{book}.md` 命名用短 slug（如 `proof_gtm0073.md`）
5. `depends` 只在 primary 节点声明，字段为 concept id

## Step 1: Python 自动生成 _manifest.json

构建脚本扫描概念目录，读取所有 YAML frontmatter，生成：

```json
{
  "id": "lagrange_theorem",
  "primary": ["theorem.md", "subgroup_def.md", "coset_def.md"],
  "tree": {
    "theorem.md": {
      "role": "theorem",
      "attachments": ["introduce.md", "proof_gtm0073.md", "proof_gtm0080.md", "exercises.md", "verify.lean"],
      "depends": { "required": ["subgroup", "coset"], "recommended": ["group_action"] }
    },
    "proof_gtm0073.md": { "role": "proof", "attaches_to": "theorem.md", "source": {"book": "gtm-0073"} },
    "proof_gtm0080.md": { "role": "proof", "attaches_to": "theorem.md", "source": {"book": "gtm-0080"} },
    "verify.lean": { "role": "lean", "attaches_to": "theorem.md" }
  },
  "cross_refs": {
    "subgroup_def.md": { "used_by": ["theorem.md"] }
  }
}
```

验证规则 (在生成时检查):
- 每个 `attaches_to` 指向的文件必须存在且为 primary 节点
- 每个 `depends.required` 必须是已存在的 concept id
- 不能有孤立文件（既不是 primary 也没有 attaches_to）

## Step 2: 完整 Pipeline

```
Step 0: OCR 逐页 JSON → stitch_chapters.py → 章节 .md
        (本地 Python, 正则匹配 CHAPTER 标记, 免费, <1min/全书)

Step 1: 章节 .md → Pro Agent → extracted/<book>/<ch>_concepts/
        Agent 输出: 多个 .md 文件 (各自带 YAML frontmatter) + .done
        (每章一个 Agent, 高并发, ¥0.80/章)

Step 2: 全部概念名列表 → Flash Agent → 去重分组
        (一次调用, ¥0.06 总计)

Step 3: 疑似重复组 → Pro Agent → merge/separate 决策
        (只在有疑点时, ¥0.03/对)

Step 4: Python 执行合并 → 机械应用 Pro Agent 决策
        (免费, 可审计, 原文件移至 _merged_archive/)

Step 5: Python 生成 _manifest.json → 扫描 frontmatter
        (免费, 验证所有引用)

Step 6: 缺失 Lean 的概念 → Pro Agent → verify.lean
        (每概念一个 Agent, Checker+Fixer v5, ¥0.02-0.08/概念)

Step 7: lake build → 验证全部 Lean
        (本地, 免费)

Step 8: Python 生成 _compiled.md → 从 _manifest.json 拼接
        (免费)
```

## Step 3: Agent Prompts

### Agent 1: Concept Extractor (Pro, thinking=true)

```
=== CONCEPT EXTRACTOR ===
Read chapter: /path/to/chapter.md

Extract ALL theorems, definitions, lemmas, propositions, corollaries, and axioms.
For each concept, create a folder and write .md files with YAML frontmatter.

OUTPUT STRUCTURE (per concept):
  concepts/<topic>/<concept_id>/
  ├── theorem.md (or definition.md, lemma.md, etc.)
  ├── introduce.md (natural language description)
  ├── proof_<book>.md (the proof from this textbook)
  └── exercises.md (if the book has exercises for this concept)

YAML FRONTMATTER RULES:
  - Every .md file MUST start with:
    ---
    role: (theorem|definition|lemma|proposition|corollary|axiom|proof|introduce|exercises)
    attaches_to: <filename>  ← REQUIRED for proof/introduce/exercises
    ---
  - theorem/definition/lemma are PRIMARY (have attachments)
  - proof/introduce/exercises are SECONDARY (must have attaches_to)
  - id: lower_snake_case_english
  - name: {en: "...", zh: "..."} if Chinese text available

FILE NAMING:
  - theorem.md, definition.md, lemma.md (one primary per concept)
  - proof_<book_slug>.md (e.g., proof_gtm0073.md)
  - introduce.md, exercises.md

TYPES to extract:
  - theorem: proved statements
  - definition: new concepts introduced
  - lemma: auxiliary results
  - proposition: important but non-central theorems
  - corollary: immediate consequences
  - axiom: assumed truths

Write .done when ALL concepts from this chapter are extracted.
Write .fail with remaining work if stuck.
```

### Agent 2: Flash Dedup Scan

```
=== DEDUP SCANNER ===
Read: /path/to/all_concept_names.json

This file contains ALL concept names extracted so far, with their English and Chinese names.
Group suspected duplicates.

OUTPUT JSON:
{
  "duplicate_groups": [
    {"canonical": "lagrange_theorem", "members": ["lagranges_theorem", "拉格朗日定理"]}
  ],
  "singletons": ["cayley_theorem", "sylow_theorem"],
  "ambiguous": [
    {"a": "fundamental_theorem_of_galois_theory", "b": "galois_correspondence",
     "reason": "may be same theorem, different naming"}
  ]
}

RULES:
- Aggressive: flag borderline cases as ambiguous
- Language variants: "拉格朗日定理" = "Lagrange's Theorem" = "lagrange_theorem"
- One-word differences ("lagrange" vs "lagranges") → duplicate
```

### Agent 3: Pro Dedup Verifier

```
=== DEDUP VERIFIER ===
Compare two concept entries that might be duplicates.

Read:
  /path/to/concept_a/theorem.md
  /path/to/concept_b/theorem.md

Decide: MERGE or SEPARATE.

If MERGE:
  - Produce merged theorem.md with both sources in depends
  - Combine proof files (rename to different proof_<book>.md)
  - Output: {"decision": "merge", "primary": "concept_a"}

If SEPARATE:
  - Output: {"decision": "separate", "reason": "..."}

Conservative: when uncertain, keep SEPARATE.
```

### Agent 4: Lean Translator (Pro, thinking=true)

```
=== LEAN TRANSLATOR (v5) ===
Translate concept to Lean 4.

Read: concept/theorem.md + concept/*_proof.md

STEPS:
1. grep ~/lean-demo/.lake/packages/mathlib/Mathlib/ -r "keyword" --include="*.lean" -l
2. Document existing Mathlib4 theorems
3. Write verify.lean with YAML frontmatter:
   ---
   role: lean
   attaches_to: theorem.md
   mathlib4_path: Mathlib.xxx.yyy
   ---
4. Compile: cd lean/MathPkg && lake env lean MathPkg/Concepts/<id>/verify.lean
5. If errors: grep → fix → recompile (up to 10 rounds)
6. Write .done when compiled (exit 0, no errors)
7. Write .fail if stuck after 10 rounds

NEVER use #check. Use example instead.
Add [Fintype X] typeclass as needed.
Only fix TYPE SIGNATURE — proof is by sorry if not yet written.
```

## Step 4: File Naming Convention

| Pattern | Example | Notes |
|---|---|---|
| Concept folder | `lagrange_theorem/` | snake_case English |
| Primary nodes | `theorem.md`, `definition.md`, `lemma.md` | One per role per concept |
| Proofs | `proof_gtm0073.md` | `<book_slug>` = series-4digit |
| Book slug | `gtm-0073`, `utm-0213`, `utx-0435`, `smm-0139` | series + 4-digit number |
| Lean | `verify.lean` | Always attaches_to primary |
| Auto-generated | `_manifest.json`, `_compiled.md` | Underscore prefix = machine |

## Step 5: Conflict Avoidance

- **Step 1 (extract)**: 每章一个 Agent, 输出到独立 chapter 目录, 永不冲突
- **Step 3 (dedup verify)**: 每对概念一个 Agent, 各自独立决策
- **Step 4 (merge)**: Python 串行执行, 先到先得
- **Step 6 (lean)**: 每概念一个 Agent, 各自写自己的 .lean 文件, 永不冲突
- 多个 Agent 同时提取到 SAME 概念 → 创建不同 proof_<book>.md → Step 3 去重时合并
